bootstrap-captcha:
Version 0.1 Created 6/6/2013		 A jQuery plug-in that along with Bootstrap and Font-Awesome will create a stylish, secure, modal validation window. Your users will appreciate the use of clear visual (randomized) icons instead of hard to read captchas. The mouse movement validation will allow developers to discover that the user is indeed a human and not a bot!
Requirements:
  • The required javascript libraries for this plug-in are:
    • jQuery
    • jQuery UI
    • Bootstrap
    • bootstrap-captcha.js (of course)
  • The required css libraries for this plug-in are:
    • jQuery UI css
    • Bootstrap-combined css
    • Font Awesome css

Note All of the javascript and css (with the exception of the bootstrap-captcha.js) are included in the examples via cdn urls. I don't believe in bloating systems with download after download of standard libraries
Initialization: 
                           $('#someDiv').bootstrapCaptcha({
                                user: 'user',
                                options: 'options',
                                here: 'here'
                           });
Examples:
  • As a Modal `Form`
    • As a document element
bootstrapCaptcha Options
Value Example Default Explanation
onDrop onDrop: function(results) Optional Function returns Object with values Object.valid (true/false) and Object.mouseUsed (true/false) Callback function triggered when the user has dropped an icon on to the target.
iconSize { iconSize: '4x'} (String) Default: '2x' Sets the size of the icons for the bootstrap captcha, any variation of the Font-awesome values are permissiable, such as '3x','4x' ect.
resetInvalidDrop { resetInvalidDrop: false } (Boolean) Default: true When a user drops an incorrect icon on the target, by default, all the icons are randomized and shuffled again to help prevent `guessing`.
resetTitle { resetTitle: 'I know you can do it!' } (String) Default: 'Oops..try again.' If you choose to set resetInvalidDrop to false you can customize the message the user receives with they drop the incorrect icon using resetTitle
textPrompt example here { textPrompt: false} (Boolean) Default: true When set to false, a copy of the correct icon, (not the default wording) prompts the user for the correct icon.
clearWhenCorrect { clearWhenCorrect: false} (Boolean) Default: true By default if the user drags the correct icon, the remaining icons and prompts disappear. You can change that by setting this to false.
displayTargetArrows { displayTargetArrows: false } (Boolean) Default: true This can turn the arrows highlighting the target on or off
